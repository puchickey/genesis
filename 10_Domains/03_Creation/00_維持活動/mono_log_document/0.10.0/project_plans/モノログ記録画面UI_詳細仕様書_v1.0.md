### 【詳細仕様書】モノログ記録画面 UI改善 (v1.0)

**更新日:** 2025年8月13日
**バージョン:** 1.0
**ドキュメントの目的:** モノログ記録画面における、操作性の向上と表現の自由度を両立させるための、新しいUI/UXの仕様を定める。

#### **1. 目的と設計思想**

本UI改善の目的は、ユーザーが日々の記録を、より直感的、快適、かつ表現豊かに行えるようにすることである。そのために、以下の設計思想を基盤とする。

*   **フォーカスと集中:** ユーザーが「今やるべきこと」に自然と集中できる、認知負荷の低いインターフェースを提供する。
*   **シームレスな体験:** 画面遷移を最小限に抑え、一連の操作が滑らかに繋がる体験を実現する。
*   **自由と制約の両立:** 感情表現の自由度を担保しつつ、UIのシンプルさと操作性を損なわない。

#### **2. UI基本設計：フォーカス・シフトUI**

モノログ記録画面の基本構造として、ユーザーの操作段階に応じてUIのフォーカスが動的に変化する「フォーカス・シフトUI」を採用する。本UIは、主に2つの状態を持つ。

*   **2.1. 状態①：感情選択モード（初期状態）**
    *   **トリガー:** ユーザーがモノログ記録画面を開いた直後。
    *   **目的:** ユーザーの意識を、記録の第一歩である「気持ちを選ぶこと」に集中させる。
    *   **画面構成:**
        *   **表示要素:** 画面上部の「切り出し画像」、中央の「感情選択コンポーネント」。
        *   **非アクティブ要素:** 「メモ入力欄」は視覚的に控えめに表示され、入力不可状態とする。「記録する」ボタンは、押せない状態（例：色が薄い）で表示する。

*   **2.2. 状態②：メモ入力モード（感情選択後）**
    *   **トリガー:** ユーザーが感情選択（第2階層まで）を完了した瞬間。
    *   **目的:** ユーザーの意識を、思考を言語化する「メモを書くこと」へスムーズに移行させる。
    *   **画面構成:**
        *   **レイアウト変化:** 「感情選択コンポーネント」は、選択結果（例：「ほっこり > のんびり」）のみをコンパクトに表示する形に縮小され、画面上部へ移動する。
        *   **アクティブ化:** 「メモ入力欄」が画面の主役となり、自動的にフォーカスが当たり、キーボードが表示される。「記録する」ボタンが押せる状態になる。

*   **【図解：フォーカス・シフトUIの状態遷移】**
    ```
    【状態①：感情選択モード】            【状態②：メモ入力モード】

    .--------------------------.         .--------------------------.
    |      (切り出し写真)      |         |      (切り出し写真)      |
    |==========================|         |==========================|
    |                          |         | [選択:ほっこり>のんびり] |
    |   ■ 気持ちを選ぶ         |         |--------------------------|
    |   (コンテンツ切り替えUI)   |         |                          |
    |   (   感情ボタン群   )   |         |   +------------------+   |
    |                          |         |   | │ メモ入力...    |   |
    |--------------------------|         |   +------------------+   |
    |   [ メモ... (非アクティブ) ] |         |                          |
    |   [ （記録する） ]         |         |   (    キーボード    )   |
    '--------------------------'         '--------------------------'
    ```

#### **3. 感情選択コンポーネント：コンテンツ切り替え方式**

限られた画面スペース内で、快適な2段階の感情選択を実現するため、「コンテンツ切り替え方式」を採用する。

*   **3.1. 第1階層の選択**
    1.  **表示:** コンポーネントのヘッダーには「気持ちを選ぶ」と表示される。ボディには、第1階層の感情ボタン群（例：「ウキウキ」「ほっこり」等）が表示される。
    2.  **操作:** ユーザーがいずれかの感情ボタンをタップする。

*   **3.2. 第2階層の選択**
    1.  **表示:** 第1階層のボタンがタップされると、コンポーネントの表示内容が、アニメーション（推奨：クロスフェード等）を伴って瞬時に切り替わる。
        *   **ヘッダー:** 「（選択した第1階層名） > 気持ちを選ぶ」というパンくずリスト形式に更新される。
        *   **ボディ:** 選択された第1階層に属する、第2階層の感情ボタン群に入れ替わる。
    2.  **操作:** ユーザーが表示された第2階層の感情ボタンをタップする。
    3.  **完了:** 選択が完了し、UI全体が「メモ入力モード」へ移行する。

*   **【図解：コンテンツ切り替え方式の操作フロー】**
    ```
    【ステップ１：第1階層を選ぶ】

    .----------------------------------.
    |   ■ 気持ちを選ぶ                 |
    |                                  |
    |      ( ウキウキ ) ( ほっこり )   |
    |      ( イライラ ) ( しょんぼり )   |
    |      ( ハッとする ) (わたしの)   |
    '----------------------------------'
        ↓ ユーザーが「ほっこり」をタップ

    【ステップ２：第2階層を選ぶ】

    .----------------------------------.
    |   ■ ほっこり > 気持ちを選ぶ     |  <-- ヘッダー更新
    |                                  |
    |      ( のんびり ) ( しみじみ )   |  <-- 中身が入れ替わる
    |      ( あったかい ) ( かわいい )   |
    |      ( やさしい ) ( なつかしい )   |
    '----------------------------------'
    ```

*   **3.3. デザイン・インタラクション詳細**
    *   **ヘッダーテキスト:**
        *   「気持ちを選ぶ」: `AppTypography.sectionHeading` を適用。色は `AppColors.inkGrey`。
        *   「（第1階層名） > 気持ちを選ぶ」: 同上。`>` の記号色も `AppColors.inkGrey` とする。
    *   **感情ボタン:**
        *   **形状:** **スタジアム形状 (`StadiumBorder`) または角丸の四角形 (`RoundedRectangleBorder`)** とする。角丸の半径は `AppColors.radiusEmotion (25.0)` を基準とし、アプリ全体のデザインと調和させる。
        *   **配色:**
            *   未選択時は、`AppColors.emotionGradients` に定義された各感情の**線形グラデーション**を背景に適用する。
            *   選択時は、グラデーションを少し濃くするか、またはボタンの周りに `AppColors.calmGreen` の枠線を表示して、選択状態を明確にする。
        *   **ラベル:** ボタン内の感情名は `AppTypography.body` を適用。色は `AppColors.inkGrey`。
        *   **タップインタラクション:** 既存の仕様書に倣い、タップした瞬間にボタンが**95%に縮小し、即座に元のサイズに戻るスケールアニメーション**（期間: 150ms）を実行する。
    *   **コンテンツの入れ替わりアニメーション:** 第1階層から第2階層への中身の入れ替わりは、**クロスフェード**（期間: 150ms）で行い、スムーズな遷移を演出する。

#### **4. 感情のカスタマイズ機能**

ユーザーが自分自身の言葉で感情を定義できるよう、感情のカスタマイズ機能を提供する。

*   **4.1. 機能要件:**
    *   ユーザーは、新しい感情の名前と、それに対応する色を自由に設定できる。
    *   作成したカスタム感情の、名前や色の編集、および削除が可能である。
*   **4.2. UI上の扱い:**
    *   **追加:** 感情管理画面や、感情選択コンポーネント内から追加フローを開始できる。
    *   **選択:** カスタム感情は、第1階層の選択肢に**「わたしの気持ち」**という専用カテゴリを設け、その中から選択できるようにする。「わたしの気持ち」をタップすると、第2階層としてユーザーが作成したカスタム感情の一覧が表示される。
    *   **管理:** 設定画面内に「感情の管理」といった専用ページを設け、そこで一覧の確認、編集、削除を行えるようにする。
