# サムネイル生成スキル（Nano Banana Pro）

## 概要

Gemini APIのネイティブ画像生成機能「Nano Banana Pro」を使用し、note.com記事のサムネイル画像を自動生成する。
既存のサムネイルと同じテイスト・品質を維持するため、背景画像・人物アイコン・参照サムネイルを渡して合成する。

## 前提条件

- `GEMINI_API_KEY` 環境変数が設定されていること
- `google-genai` と `Pillow` がPythonにインストールされていること
- 以下の素材が存在すること:
  - 背景画像: `note/thumnail/thumnail_background.jpg`
  - 人物アイコン: `note/thumnail/icon_note/` 配下の画像
  - 参照サムネイル: `note/thumnail/article/` 配下の既存サムネイル

## 素材パス

```
G:\マイドライブ\Genesis_OS\10_Domains\03_Creation\01_投資活動\note\
├── thumnail/
│   ├── thumnail_background.jpg       ← 共通背景
│   ├── icon_note/                    ← 人物アイコン（6種）
│   │   ├── icon_笑顔で指を立てる.png  ← ポジティブ系
│   │   ├── icon_正面.png             ← 落ち着き系
│   │   ├── icon_右手を差し出す.png    ← 提案系
│   │   ├── icon_左手を差し出す.png    ← 案内系
│   │   ├── icon_指先でつまむ.png      ← コツ・ポイント系
│   │   └── icon_頭を抱えて驚く.png    ← 衝撃・驚き系
│   └── article/                      ← 過去サムネイル（スタイル参照用）
└── thumbnails/                       ← 生成結果の出力先
```

## 実行スクリプト

`G:\マイドライブ\Genesis_OS\.agent\scripts\thumbnail_generator.py`

### 基本的な使い方

```powershell
# 環境変数を設定
$env:GEMINI_API_KEY = "<YOUR_API_KEY>"

# 全件生成
python thumbnail_generator.py

# 番号指定で個別生成(1, 2, 3...)
python thumbnail_generator.py 3
```

### サムネイルの追加・変更

`thumbnail_generator.py` 内の `THUMBNAILS` リストに辞書を追加する。

```python
{
    "top_text": "上段テキスト",        # 白帯の上。4〜8文字
    "main_text": "メインワード",       # メタリック大文字。3〜6文字（最重要）
    "bottom_text": "下段テキスト",     # 白文字。5〜10文字
    "icon_name": "icon_笑顔で指を立てる.png",  # 人物ポーズ
    "reference_article": "note_023.png",       # スタイル参照元
    "output_name": "thumb_xxx.png",            # 出力ファイル名
}
```

## レイアウトルール

サムネイルは「左テキスト＋右キャラクター」の2カラム構成。

```
┌──────────────────────────────────────────┐
│ ┌──────────┐                             │
│ │ 上段(白帯) │          [人物アイコン]     │
│ └──────────┘          （右半分に配置）    │
│                                          │
│  メインテキスト（メタリック大文字）         │
│                                          │
│  下段テキスト（白太字）                    │
└──────────────────────────────────────────┘
```

### テキスト3層

| 層 | 文字数 | スタイル | 役割 |
|---|--------|---------|------|
| 上段 | 4〜8文字 | 白帯＋黒/ダーク文字 | フック・キャッチ |
| 中段 | 3〜6文字 | メタリック/白・超大文字 | メインキーワード |
| 下段 | 5〜10文字 | 白太字 | 補足・ブランド |

### アイコン選定基準

| 記事の雰囲気 | 推奨アイコン |
|------------|------------|
| ポジティブ・推し | `icon_笑顔で指を立てる.png` |
| 落ち着き・共感 | `icon_正面.png` |
| 提案・おすすめ | `icon_右手を差し出す.png` |
| 案内・こちらへ | `icon_左手を差し出す.png` |
| コツ・ポイント | `icon_指先でつまむ.png` |
| 衝撃・驚き・不安 | `icon_頭を抱えて驚く.png` |

## 品質チェックリスト

生成後に以下を確認する：

- [ ] アスペクト比が横長（正方形でない）
- [ ] テキスト3行が全て正確に表示されている
- [ ] 文字が省略・誤字されていない
- [ ] キャラクターとテキストが重なっていない
- [ ] キャラクターの描画に異常がない（手が白い、顔が崩れる等）
- [ ] 背景の基板パターンが見えている
- [ ] 白枠ボーダーがある
- [ ] パーティクル効果がある

## 画像ルール（必須）

以下はサムネイル生成時に必ず守るルール。プロンプトにも反映済み。

### 人物キャラクターの肌色
- キャラクターの肌色は**元のアイコン画像の自然な肌色を正確に維持**すること
- 背景のティール色に引っ張られて**青白くしない**
- 背景色で**染めない**

### 「Google」のテキスト表示
- テキストに「Google」という単語が含まれる場合、**本物のGoogleロゴと同じカラフルな配色**で表示すること
- 色の順番: G(青) o(赤) o(黄) g(青) l(緑) e(赤)
- これは既存サムネイル note_022.png でも採用されているスタイル

## 注意事項

- **画像はアーティファクトでは表示できない**。確認時はファイルリンクで提示すること
- 生成結果に異常があれば、同じ番号で再実行すれば上書きされる
- モデル: `gemini-3-pro-image-preview`（Nano Banana Pro）を使用
- 1枚あたりの生成時間: 約30〜60秒
