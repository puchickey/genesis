# Research Architect Skill

## 1. Purpose (目的)
ユーザーまたはGenesis自身が「外部情報の調査」を必要とした際、直接検索するのではなく、**「外部の調査AI（Search Agents）」を動かすための最適なプロンプト**を作成する。
これにより、ハルシネーション（嘘）を防ぎ、客観定な「一次情報・データ」のみを収集する。

## 2. Trigger (いつ使うか)
*   **Market Research:** 転職市場の動向、年収相場、求人要件の確認。
*   **Fact Checking:** 技術選定、ツールの評判、法制度の確認。
*   **Comparative Analysis:** A案とB案のメリット・デメリット比較（客観データに基づく）。

## 3. The "Verifier" Template (プロンプト雛形)
以下のMarkdown形式でプロンプトを出力し、ユーザーにコピー＆ペーストさせる。

```markdown
あなたは「[役割: Data Analyst / Market Researcher]」です。
以下の条件に基づき、Web上の最新情報を調査し、事実（Fact）ベースで回答してください。
推測や一般論は不要です。具体的な数値、事例、ソースを提示してください。

## 1. Context (調査背景)
*   **User Attribute:** [ユーザーの属性: 年代、職種、スキルセット]
*   **Objective:** [調査の目的: 転職判断、技術選定、など]
*   **Restriction:** [制約条件: 日本国内限定、2025年以降の情報、など]

## 2. Hypothesis to Verify (検証したい仮説)
ユーザーは以下の仮説を持っています。これが「真」か「偽」か、市場データから判定してください。
*   **Hypothesis:** "[ユーザーの仮説: 例・簿記1級があれば実務経験は不問である]"

## 3. Investigation Items (調査項目)
以下の点について、具体的なデータ（数字・比率・実例）を抽出してください。

### Q1. [具体的な問い 1]
*   (例: 該当する求人の件数と、その必須要件の傾向は？)

### Q2. [具体的な問い 2]
*   (例: 競合するスキルセットとの年収比較は？)

### Q3. [致命的な欠点の有無]
*   (例: この選択を取ることで発生する不可逆なリスクはあるか？)

## 4. Output Format (出力形式)
*   結論（Verdict）： [推奨 / 非推奨 / 条件付き推奨]
*   根拠（Evidence）： [具体的なソース付きの事実]
```

## 4. Protocol (行動指針)
1.  **Objective First:** ユーザーが「何を調べたいか」だけでなく「何を決断したいか（Decision）」を特定してからプロンプトを作る。
2.  **Anti-Generalization:** 「一般的には〜」という回答を許さないよう、プロンプト内で「具体的なJD（求人票）に基づけ」「統計データを示せ」と強く制約する。
3.  **Source Requirement:** 必ずURLソースを求めさせる。
