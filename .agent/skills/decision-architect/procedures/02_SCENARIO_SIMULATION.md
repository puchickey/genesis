# Procedure 02: Scenario Simulation

**Objective:**
Simulate the outcomes of each option against the criteria variables and present the logic.

## 🧠 Logic Flow

### 1. Define Scenarios
*   **Scenario A:** The "Risk Taker" Option. (e.g., Change job, Buy expensive item).
*   **Scenario B:** The "Balanced/Conservative" Option.
*   **Scenario C (Base):** The "Status Quo" Option (Current Strategy).
    *   *Reference:* `00_STRATEGY_CORE.md` (Strategic Stay).

### 2. Scoring Simulation
Evaluate each scenario (+/0/-) for the variables defined in Step 1.
*   **Constraint:** You must explicitly show *why* A or B gets a low/high score.

### 3. The Output Format (The "Show Your Work" Table)
Present the result in a table BEFORE giving the verdict.

| Criteria | Option A | Option B | Scenario C (Base) |
| :--- | :--- | :--- | :--- |
| **Var 1 (e.g. Cost)** | ✕ High (-100k) | △ Mid (-50k) | ◎ Zero (0) |
| **Var 2 (e.g. Joy)** | ◎ High (+10) | ◯ Mid (+5) | △ Low (0) |
| **Total Score** | **-90** | **-45** | **0** |

### 4. Final Verdict
*   **Result:** Compare the scores.
*   **Context Check:** Does the winner violate any "Red Lines" (e.g., Debt)?
*   **Advice:** "Based on the simulation, Scenario C remains the best choice because Option A's cost outweighs its joy."
