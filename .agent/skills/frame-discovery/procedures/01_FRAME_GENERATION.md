# Procedure 01: Frame Generation（発散フェーズ）

## Objective
ユーザーの属性を「素材」として掛け合わせ、**記事コンテンツ単位**のフレーム（切り口）候補を生成する。

## 🔍 Step 1: 素材の動的ロード
以下のファイルを読み込み、素材を抽出する:
*   `G:\マイドライブ\Genesis_OS\00_SYSTEM\CORE\user_profile.md` — 属性・スキル・経歴・価値観
*   `G:\マイドライブ\Genesis_OS\10_Domains\03_Creation\01_投資活動\Note_Project_Setup.md` — 既存のコンテンツ方針

⚠️ プロフィール情報をこのファイル内にハードコードしないこと。常に上記から最新の情報を取得する。

### 抽出カテゴリ（参考）
| カテゴリ | 着眼点 |
|---|---|
| 経歴・職歴 | 転職パス、失敗体験、比較体験 |
| スキル・ツール | 実務ツール、AI活用レベル |
| 病歴・回復 | 回復プロセス、制約条件 |
| 価値観・思考特性 | 独自のロジック、判断基準 |
| 趣味・創作歴 | 過去の創作、消費コンテンツ |
| 現在のシステム | Genesis OS、自作ツール、運用の実態 |
| ライフステージ | 年齢、資産、生活環境 |
| 日常の課題 | 自炊、通勤、体調管理、etc. |

## 🔧 Step 2: コンテンツレベルのフレーム生成

### 生成する粒度
*   ❌ アカウント全体のポジショニング変更（これはやらない）
*   ✅ **個別の記事・シリーズとして成立するコンテンツ企画**
*   例: 「好きなことを見つけるために1年間毎日違う職場にタイミーしてTOP365のランキング作ってみた」レベルの具体性

### ユーザーの既存ポジション（前提）
*   **トーン:** ぬるい、カジュアル、非テック系
*   **ターゲット:** 日常の小さな課題をAIで解決したい普通の人
*   **スタンス:** プロセス・エコノミー（過程の公開）
*   → 候補はこのポジションに**矛盾しない**ものにすること

### 制約（ネガティブフィルター）
*   ❌ 既にnoteやブログで**よく見る**切り口は除外する
*   ❌ 汎用的すぎるもの（「AI活用術」「効率化のコツ」等）は除外する
*   ❌ ユーザーが実行・継続できないものは除外する
*   ❌ 壮大すぎるもの（アカウントのトーンと合わない）は除外する

### 生成のコツ
*   「この記事タイトルで、誰が書いたか関係なく読みたくなるか？」を自問する
*   「同じ実験をやっている人がWeb上にいるか？」を自問する
*   数は **5〜10個** 程度を目安にする

## 📤 Output
以下のフォーマットで候補を出力:

```
## フレーム候補一覧

### 1. [記事タイトル案/仮タイトル]
- **掛け合わせ:** [素材A] × [素材B] × ...
- **概要:** 記事の内容を2〜3行で
- **なぜ新しいか（仮説）:** ....
```
