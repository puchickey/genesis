# Procedure 03: Evaluation（収束フェーズ）

## Objective
調査結果をもとに、フレーム候補の独自性を判定する。

## 📥 Input
ユーザーから共有される調査AI（Perplexity/Genspark等）の調査結果。

## 🔍 Evaluation Matrix

各候補について以下の3軸で評価する:

| 軸 | 判定基準 | 合格ライン |
|---|---|---|
| **①新規性** | 同じ掛け合わせ/切り口で継続発信している人がいるか | **いない**、または**極めて少ない** |
| **②一人歩き力** | 「誰が書いたか」を伏せても、タイトルだけで読みたくなるか | ユーザーが「Yes」と判断 |
| **③実行可能性** | ユーザーが継続的にコンテンツを出せるか | ユーザーが「Yes」と判断 |

*   ①はデータ（調査結果）で判定
*   ②③はユーザーの主観で判定（AIは判断しない）

## 📤 Output

```markdown
## フレーム評価結果

### [フレーム名]
| 軸 | 結果 | 根拠 |
|---|---|---|
| ①新規性 | ✅/⚠️/❌ | 調査結果の要約 |
| ②一人歩き力 | （ユーザー判断） | — |
| ③実行可能性 | （ユーザー判断） | — |

**Verdict:** ADOPT / RETRY / DROP
```

## 🔄 Loop / Exit

*   **ADOPT:** フレーム確定。結果を `Note_Project_Setup.md` 等に反映して終了。
*   **RETRY:** 素材の掛け合わせを変えて Phase 1 へ戻る。
    > Load: `procedures/01_FRAME_GENERATION.md`
*   **DROP:** その候補を破棄し、残りの候補を評価。全滅なら Phase 1 へ。
