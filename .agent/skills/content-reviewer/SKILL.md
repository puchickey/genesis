# Content Reviewer

記事コンテンツをCARフレームワーク（Paul Smith提唱）に基づいてレビューし、改善点をフィードバックする。

## CARフレームワーク概要

人間の脳は論理的情報より「物語」として提示された情報を記憶しやすい。CARはストーリーを3要素で構成する型。

| 要素 | 内容 | ポイント |
|---|---|---|
| **C (Context)** | 背景情報：人物・時間・場所・状況 | 読者が「なぜこの話が自分に関係あるか」を理解できること |
| **A (Action)** | 中心的な行動：課題に対して何をしたか | 感情的変化や葛藤を含み、読者がイメージできる具体性 |
| **R (Result)** | 結果 + 教訓：何が起き、何を学べるか | 「だから〇〇が重要だ」という読者が活かせる示唆 |

## レビュー手順

1. ユーザーから記事テキストを受け取る
2. 以下のチェックリストで各項目を評価（✅ / ⚠️ / ❌）
3. 各項目にコメントと改善提案を付ける
4. 総合評価とNext Actionを提示

## チェックリスト

### C (Context：文脈)
- [ ] 読者が全体像を理解するための背景情報（人物・時間・場所・状況）が十分か
- [ ] 登場人物の行動の動機・感情の根拠が文脈から明確に伝わるか
- [ ] 読者がこの話に関心を持つべき理由が文脈から理解できるか

### A (Action：行動)
- [ ] 中心的な出来事・課題・具体的な行動が明確に語られているか
- [ ] 読者がイメージできる具体性があるか（感情的変化・葛藤を含む）
- [ ] 行動がContextで示された課題・目標と関連しているか

### R (Result：結果)
- [ ] 行動の結果として何が起きたかが明確か
- [ ] 読者が学べる教訓・メッセージが含まれているか
- [ ] 教訓が読者自身の経験や未来に活かせる形で提示されているか
- [ ] 結果と教訓がストーリー全体と一貫しており不自然さがないか

## 出力フォーマット

```

```
