## 📂 Genesis OS File Rules (整理の哲学)
1.  **投資活動 (Investment) = "Active Transformation"**
    *   **定義**: 現状のステータスを「書き換える」ための能動的な活動。
    *   **特徴**: 新規スキルの習得、キャリアチェンジ、新しい仕組みの構築。
    *   *例: 転職活動、新規アプリ開発、婚活*
2.  **維持活動 (Maintenance) = "Asset & Routine"**
    *   **定義**: 過去の活動の「記録/資産」および、自分を保つための「ルーティン」。
    *   **特徴**: 既に持っているスキルの記録、趣味、完了した学習ログ。
    *   *例: 簿記(取得済)、ポーカー(趣味)、家計簿*
3.  **動的な運用 (Life Cycle)**
    *   活動フェーズに合わせてフォルダ間を**移動**させる。
    *   *例: 簿記の勉強を再開してキャリアアップを狙う → 「維持」から「投資」へ移動*
4.  **Dynamic Flow (諸行無常)**
    *   **Rule**: 活動のフェーズに応じて、ファイルを素早く移動させる。
        *   **Start**: `01_投資活動` (New Project)
        *   **Succeed**: `00_維持活動` (Becomes Asset/Routine)
        *   **Finish/Drop**: `99_アーカイブ` (Log/History)
    *   **Why**: 「今、何に集中すべきか」をフォルダ構造で物理的に可視化するため。過去の遺産が現在の視界を邪魔しないようにする。
