# 🧬 Genesis Hustle Protocol v7.0: The Sequential Procedure

本プロトコルは、副業の「発想」から「資産化」までの全工程を、**一筆書きのシーケンス（手順）**として定義したものです。
ユーザーは、現在の自分が「どの番号（SEQ-ID）」にいるかを確認し、その指示に従って機械的に行動してください。

---

## 🏗️ Phase 1: Foundation Soil (土台と種)
*目的: 「負けない環境」を作り、「勝てる種」を選別する。*

### [SEQ-001] Define Purpose (目的定義)
*   **Action:** 以下の問いに対し、明確な言語化を行う。
    *   なぜやるのか？（資産性 / スキル獲得 / 衝動）
    *   **Check:** 「小遣い稼ぎ」になっていないか？（なっていれば中止）

### [SEQ-002] Secure Environment (環境確保)
*   **Action:** 物理的・時間的リソースを確保する。
    *   場所: 集中できる聖域（カフェ/書斎）を確保した。
    *   時間: 週間スケジュールに「10時間」のブロックを天引きした。
    *   **Check:** 本業の時間や睡眠時間を削っていないか？

### [SEQ-003] Gap Analysis (本業診断)
*   **Action:** 本業の「徳（Virtues）」と「悪（Evils）」をリストアップする。
    *   **Check:** 本業で欠けている「徳」は何か？（これが副業のミッションになる）
    *   **Check:** 本業で既に抱えている「悪」は何か？（これを副業で反復してはならない）

### [SEQ-004] Ideation via SCAMPER (アイデア量産)
*   **Action:** ミッションを満たすアイデアを強制的に10個出す。
    *   SCAMPER: 代用、結合（スキルxスキル）、適応、拡大/縮小、転用、削除、逆転。
    *   **Check:** 10個出揃ったか？（質より量）

### [SEQ-005] The Filter Matrix (一次審査)
*   **Action:** 出したアイデアを「15の要素」でフィルタリングする。
    *   **Check:** 本業の「悪」を悪化させる要素（例: 更なる長時間拘束）はないか？（あれば即廃棄）
    *   **Check:** 本業の「徳」の欠落を埋める要素はあるか？

### [SEQ-006] The 5 Conditions (二次審査)
*   **Action:** 残った候補を「副業の科学・5つの条件」でチェックする。
    *   1.自律性 / 2.完結性 / 3.フィードバック / 4.意義 / 5.**多様性**
    *   **Check:** 特に「多様性（スキル資産化）」はあるか？

## 🎰 Phase 2: The Bandit Run (実験)
*目的: 最小コストで「当たり判定」を行う。*

### [SEQ-007] Portfolio Selection (アーム選定)
*   **Action:** 審査を通過した候補を「インパクト×コスト」マトリックスに配置する。
    *   **Check:** 「High Impact (収益/資産) × Low Cost (手軽)」の領域にあるか？
    *   **Action:** 最上位の1〜3個を「実験対象（アーム）」として認定する。

### [SEQ-008] Pretotyping (プレトタイプ作成)
*   **Action:** アームごとの「最小試作品」を作る。
    *   内容は「LP」「告知文」「提案書」のみ。完成品は作らない。
    *   **Check:** 制作に3日以上かけていないか？（時間はコストである）

### [SEQ-009] KPI Definition (勝利条件設定)
*   **Action:** 売上以外の「先行指標」を設定する。
    *   指標例: 継続率、アクティブ率、問い合わせ数。
    *   **Check:** 「PV」などの虚栄の指標になっていないか？

### [SEQ-010] Micro-Sprint (実験開始)
*   **Action:** 期間（1週間）または予算（1万円）を区切り、市場に投下する。
    *   **Check:** 本業の業務時間中に手を出していないか？（Strict Rule）

---

## ⚖️ Phase 3: The Bayesian Judge (検証と学習)
*目的: データだけを見て、進むか引くかを決める。*

### [SEQ-011] Data Collection (結果計測)
*   **Action:** [SEQ-009]で設定したKPIの実数を確認する。
    *   **Check:** 事前に決めた「ボーダーライン」を超えたか？

### [SEQ-012] The Verdict (判決)
*   **Action:** 結果に基づき、以下の3つから機械的に選択する。
    *   **Case A (Win):** ボーダー超え → [SEQ-015]へ。
    *   **Case B (Lose):** ボーダー未満 → [SEQ-013]へ。
    *   **Case C (Draw):** 判断不能 → 条件を変えてもう一度 [SEQ-010]へ。

### [SEQ-013] Post-Mortem Analysis (敗因分析)
*   **Action:** （Case Bのみ）なぜダメだったか言語化する。
    *   原因: ニーズなし？ ターゲット違い？ 訴求弱すぎ？
    *   **Check:** この学びをメモに残したか？（これが資産になる）

### [SEQ-014] The Drop (撤退)
*   **Action:** 感情を排してプロジェクトを完全停止・廃棄する。
    *   **Check:** 「もったいない」と思ってダラダラ残していないか？
    *   -> [SEQ-004]に戻り、次のアイデアへ。

---

## 📈 Phase 4: Scaling (資産化)
*目的: 勝てる戦いだけを、自動で回し続ける。*

### [SEQ-015] Concentration (集中投資)
*   **Action:** 他の実験を一時停止し、リソースの90%をこのWin案件に注ぐ。
    *   **Check:** ここで初めて「売上（遅行指標）」をKPIに格上げする。

### [SEQ-016] SOP Creation (仕組み化)
*   **Action:** 成功した手順をマニュアル化、またはツール化する。
    *   **Check:** 自分以外の人間（または来週の疲れた自分）でも再現できるか？

### [SEQ-017] Resource Fixation (配分固定)
*   **Action:** 「70% (Win運用) : 20% (次のBandit) : 10% (Review)」の比率を確立する。

---

## 🔄 Phase 5: The Ritual (運用)
*目的: このサイクルを永遠に回す。*

### [SEQ-018] Weekly Review (週次戦略会議)
*   **Action:** 毎週日曜 AM10:00、自身と会議を行う。
    *   議題: 今、自分は [SEQ-001] 〜 [SEQ-017] のどこにいるか？
    *   議題: 次の1週間で、どの番号に進むか？
